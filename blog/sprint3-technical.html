Explain control flow and loops using an example process from everyday life.

Control flow can be explained as the process in which an order of statements are executed in a program. The best example of control flow being executed is in a loop. As the program is executed, it flows from the first statement and loops through until the last statement, unless it is told to stop. This is called flow of control.

If we apply control flow and looping into a process from everyday life, showering would be a prime example. You have a set of steps for showering which are in a specfic order.

You start by turning on the shower, washing your body, shampoo and conditioning your hair and turning off the water. These are the steps which are stated as the flow of control.

When we add loops into this process, it would basically be like moving through each step of the control flow until we cannot complete anymore steps. The last step in the process of showering would be turning off the water. If for some reason you had run out of shampoo/conditioner, you would skip this step and end in turning off the water which is the final step.

Describe what the DOM is and an example of how you might interact with it.

The DOM (Document Object Model) is basically a tree of nodes(objects) that make up your webpage. You can think of it as a family tree in a way, you have families of nodes that keep branching off into other families of nodes. Within your tree, you have your coding languages; HTML, CSS and JavaScript. The DOM acts as a glue to hold all of these together and allows them to function with each other.

One way to interact with the DOM, would be through the developer tools in your web browser. By accessing this, you could manipulate the DOM by entering some JavaScript entries into the console. This will visually interact with your webpage, creating dynamic HTML. As an example, you might have a button within your HTML that doesn't really do anything exciting. You then add some script that will show a counter that increases everytime it is clicked. This is done by manipulating the DOM with JavaScript, creating a more functionable webpage.

Explain the difference between accessing data from arrays and objects.

An object is a data type which includes one or more properties and values. The object will be named something that relates to the contents. Lets say we want to create an object that gives us some information about a specific person. This is basically how it would look:

var person = {name: "Jack", age: 24, city: "Wellington"};

When you break this down, you have the object which is defined as a variable(var) and called 'person'. As you enter the contents of the object, you have your properties (name, age and city) and values ("Jack", 24 and "Wellington").

To access information in this object, we can do this by using dot-notation (.) or bracket-notation ([]).

Dot-notation would look like this: console.log(person.name); - this then returns the value of "Jack".
Bracket-notation on the other hand would look like this: console.log(person["age"]); - which would return the value of 24.

If you need to access a property that has more than one word with a space in between, you would have to use the bracket-notation, as dot-notation only supports whole words.

An array is a collection of elements that are stored within [] notations. These apply to a variable, very similar to an object. An array would look something like this:

var cities = ["Wellington", "Auckland", "Christchurch"];

As you can see, there is a variable(var) applied called 'cities'. These elements each have an index number already applied to them, so to access these we use the zero based index system.

In a similar fashion to accessing objects, we use the variable name. But instead of entering a specfic value, we enter the index number:

console.log(cities[0]); - this will return the value of "Wellington", as our index begins at 0.

Explain what functions are and why they are useful.

A function is a section of code written in JavaScript, designed to be able to perform a specfic set of tasks. Functions will usually be written with a list of statements to be able to perform an output of your choosing. Anything that is located within a function is classed as a 'local scope', which means they cannot be accessed from outside the function. The function can access variables 'classed as global scopes' within the script though and any function within itself.

Functions are incredibly useful as they are recyclable, meaning you can use them as many times as you'd like. Once a function has been created it will permanently stay in your script, allowing you to use it over and over again. Having the ability to write multiple functions also allows for less complexity and better organisation.
